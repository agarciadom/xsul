This sample demonstrates WSDL "contract first" approach to accessing and creating Web Services.

Contract: WSDL

First WSDL file needs to be created or obtained from repository. We used well known global weather sample found in CapeScience.

Client side

Dynamic Invocations

WSDL file may be used directly theough Super Dynamic Invoker that allows to invoke service described in WSDL without need to generate Java stub. Example (before running make sure to set CLASSPATH by using classpath.[sh|bat]): 

java [-Ddebug] xsul_dii.XsulDynamicInvoker samples/simplesoap/contract/GlobalWeather.wsdl  {GlobalWeather}getWeatherReport JFK

Invocations using stub (RPC style)

To allow more type-safe invocations Java interface needs to be generated: WSDL file is compiled by xwsldc to generate Java interface. This interface then can be used to create stub to interact with remote service.

Invocations using stub (MSG style)

Generated stub allows to send XML Infoset representing message (using XB1,DOM API, or any other XML API with additional transformations).

Web / XML Services Utility Library 1.1

Quick introduction

This is a research vehicle to get a set of modules that are easy to assemble to create Web XML based services.

However as this is research project so it is changing and backward compatibility is not top priority. Still you have source code and can keep it :-)

That means that users input is very important and can very much shape where this library is going to make it utility library you use: http://www.extreme.indiana.edu/xgws/xsul/

 

How to get started

Build: ant

Clean build: ant all

Run sample: first set your CLASSPATH
(t)csh: source classpath.csh
bash: source classpath.sh
Windows: classpath.bat

When CLASSPATH is set we can run one of samples, the simplest will start in-process service (mini HTTP server), call it few times to say hello, and quit (examine source code in src/samples). Try java xsul_sample_hello.Run

To see debug output use logger:

  • sample related logging output: java -Dlog=xsul_sample_hello:ALL xsul_sample_hello.Run
  • WS/XSUL runtime and sample logging: java -Dlog=xsul:ALL,xsul_sample_hello:ALL xsul_sample_hello.Run
  • all logging (LOT OF OUTPUT!): java -Dlog=:ALL xsul_sample_hello.Run

Then make sure to read User's Guide and list of changes.

Modules

  • common (no dependencies except for XML pull parsing API - currently uses XPP3/XB1 for XML Infoset API)
  • http_client (depends on common)
    - provides basic HTTP 1.x client side support with simple HTTP connection manager
  • http_server (depends on common)
    - provide basic HTTP 1.x server side support with simplified Servlet-like API
  • invoker* (depends on http_client)
    - allows simple invocation of service by sending XML on top of HTTP
  • processor* (depends on http_server)
    - provides basic support for processing incoming XML over HTTP
  • dsig_globus (depends on common and security jar file from lib/dsig*)
    - provides XML digital signature and limited WS SEC support using Globus Toolkit 3 grid proxy certificates
  • soap_util, soap11_util and soap12_util - set of utility classes to facilitate manipulating
    both SOAP 1.1 and 1.2 messages
  • soap11, soap12, soap_factory: set of nice interfaces to manipulate SOAP messages
    independent of SOAP version (1.1/1.2) in OO way that allows extensibility
  • epr_router (depends on rpc, XmlBeans: lib/xbeans)
    - Routes incoming messages based on WSA EPR to specific XBeans document dispatcher
  • xbeans_document_dispatcher (depends on rpc, lib/xbeans)
    - Dispatches messages with specific QName to a mapped method in a java target. Mapped
    methods can have XmlElement or XmlBean Object as parameter and response docs.
  • xbeans_document_invoker (depends on rpc, lib/xbeans)
    - invokes a method in remote service using doc style invocation, taking and returning XmlObjects.
  • xbeans_type_handler (rpc, lib/xbeans)
    - serializes and deserializes XmlBean objects
  • xwsdl small footprint WSDL impl
    - lightweight API to work with WSDL 1.1
  • xwsif* (depends on xwsdl) dynamic invocations using WSDL
    - recasting of Apache WSIF 2.0 interfaces to use xwsdl
  • xwsif_provider_xsul* (depends on xwsdl and invoker)
